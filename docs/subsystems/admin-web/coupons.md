# 管理后台 — 优惠券管理

上级文档：[管理后台 Web](../admin-web)

---

## 模块概述

优惠券管理包含两个维度：**优惠券模板管理**（定义券的规则）和**优惠券实例管理**（查看每张券的发放、使用状态）。管理后台提供创建模板、批量发放、查看核销记录等能力。

---

## 页面路由

| 路由 | 页面 | 说明 |
|---|---|---|
| `/coupons` | 优惠券模板列表 | 所有优惠券模板，含发放/核销统计 |
| `/coupons/new` | 新增优惠券模板 | 创建新的优惠券模板 |
| `/coupons/:id/edit` | 编辑优惠券模板 | 编辑已有模板（部分字段不可改） |
| `/coupons/:id/records` | 发放与核销记录 | 该模板下所有券的使用情况 |

## 优惠券模板列表

### 列表字段定义

| 列名 | 字段 key | 类型 | 说明 |
|---|---|---|---|
| 模板 ID | `id` | String | 系统唯一标识 |
| 券名称 | `name` | String | 如「新用户专享 50 元券」「限时 9 折券」 |
| 券类型 | `type` | Enum | 折扣券 / 满减券 / 固定金额券 / 免费体验券 |
| 优惠内容 | `discountDesc` | String | 根据类型展示：如「9 折」「满 100 减 20」「抵扣 50 元」「免费兑换体验卡」 |
| 适用产品 | `applicableProducts` | String[] | `全场通用` 或具体产品名称列表 |
| 使用门槛 | `thresholdDesc` | String | `无门槛` 或 `满 N 元可用` |
| 活动时间 | `startAt` ~ `endAt` | DateTime | 领取窗口期；未设时间则显示 `长期有效` |
| 有效天数 | `validDays` | Int | 领取后 N 天内有效 |
| 总量/已发/已用 | 统计 | — | `200 / 150 / 89` 格式展示 |
| 核销率 | — | String | `已用/已发` 百分比，如 `59.3%` |
| 状态 | `isActive` | Boolean | 启用 / 已停用 |

### 筛选条件

| 筛选项 | 类型 | 说明 |
|---|---|---|
| 关键词 | 输入框 | 券名称模糊搜索 |
| 券类型 | 多选 | 折扣券 / 满减券 / 固定金额券 / 免费体验券 |
| 状态 | 单选 | 全部 / 启用 / 已停用 |

## 新增 / 编辑优惠券模板

### 表单字段

#### 基本信息

| 字段 | 组件 | 必填 | 校验规则 | 说明 |
|---|---|---|---|---|
| 券名称 | 文本输入 | ✅ | 2-30 字符 | 如「新用户专享 50 元券」 |
| 券类型 | 单选 | ✅ | 枚举值 | 折扣券 / 满减券 / 固定金额券 / 免费体验券 |

#### 优惠规则（根据类型动态显示）

| 字段 | 组件 | 条件 | 校验规则 | 说明 |
|---|---|---|---|---|
| 折扣率 | 数字输入 | 类型=折扣券 | 0.01-0.99，如 0.9 = 9 折 | — |
| 优惠金额 | 数字输入 | 类型=满减券/固定金额券 | ≥ 0.01，2 位小数 | — |
| 使用门槛 | 数字输入 | 类型=满减券 | ≥ 优惠金额 | 满 N 元可用 |
| 兑换产品 | 下拉选择 | 类型=免费体验券 | 必选 | 选择一个已上架的产品 |

#### 使用范围

| 字段 | 组件 | 必填 | 说明 |
|---|---|---|---|
| 适用产品 | 多选 | ✅ | 不选=全场通用；可选择具体产品 |
| 使用门槛 | 数字输入 | ❌ | `满 N 元可用`；留空=无门槛（免费体验券默认无门槛） |

#### 发放规则

| 字段 | 组件 | 必填 | 校验规则 | 说明 |
|---|---|---|---|---|
| 总发放数量 | 数字输入 | ❌ | ≥ 1；留空=不限量 | — |
| 活动开始时间 | 日期时间 | ❌ | — | 不设=立即生效 |
| 活动结束时间 | 日期时间 | ❌ | ≥ 开始时间 | 不设=长期有效 |
| 领取后有效天数 | 数字输入 | ✅ | ≥ 1 | [建议值] 默认 7 天 |

#### 高级设置

| 字段 | 组件 | 必填 | 说明 |
|---|---|---|---|
| 每人限领 | 数字输入 | ❌ | 默认 1 张；0=不限 |
| 注册自动发放 | 开关 | ✅ | 开启后新用户注册时自动获得 1 张 |
| 购买赠券 | 下拉 | ❌ | 选择一个产品，购买该产品时赠送此券 |

> **[建议值]** 注册自动发放默认关闭，需运营手动开启并配置具体券。

### 编辑限制

- **券类型**创建后不可修改（影响已有券的计算逻辑）
- **折扣率/优惠金额**修改后不影响已发放的券（券实例保存时已固化优惠值）
- **停用**后不可重新启用（避免逻辑混乱），需新建模板

### 手动发放（给单个用户发券）

在用户详情页中提供「发放优惠券」操作：

1. 选择优惠券模板
2. 确认发放
3. 用户立即获得一张券

## 手动发放（用户详情入口）

管理后台用户详情页中增加「发放优惠券」操作，供客服/管理员处理投诉或补偿时使用。

### 交互流程

1. 在用户详情页点击「发放优惠券」
2. 弹出优惠券模板选择器（仅显示当前启用的模板）
3. 选择模板后显示券详情预览
4. 确认发放，写入发放记录
5. 操作日志记录：操作人、发放原因（可选填写）

## 用户故事

### US-COUPON-01：运营人员创建新优惠券活动

> 作为运营人员，我想创建一张限时满减优惠券，设定活动时间、使用门槛和发放数量，以便在促销活动中使用。

**验收标准**：
- [ ] 可选择券类型，根据类型动态切换优惠规则表单
- [ ] 可设置活动开始/结束时间，过期后自动停用
- [ ] 可设置总发放数量，达到上限后不再发放
- [ ] 可设置适用产品范围
- [ ] 保存后立即出现在模板列表中

### US-COUPON-02：管理员批量发放优惠券

> 作为管理员，我想给符合条件的用户批量发放优惠券，以便开展促销活动。

**验收标准**：
- [ ] 支持按「全部用户」「指定用户」「条件筛选」三种方式选择发放对象
- [ ] 发放前显示预计发放数量和剩余可发数量
- [ ] 发放操作需二次确认
- [ ] 发放完成后在记录页可查看所有发放记录

### US-COUPON-03：客服给投诉用户手动发放补偿券

> 作为客服，我想给投诉的用户手动发放一张优惠券作为补偿。

**验收标准**：
- [ ] 在用户详情页有「发放优惠券」入口
- [ ] 可选择当前启用的优惠券模板
- [ ] 发放后用户立即获得，可在小程序中看到
- [ ] 操作记录写入操作日志

### US-COUPON-04：运营人员查看优惠券核销情况

> 作为运营人员，我想查看某张优惠券的核销率和使用详情，以便评估活动效果。

**验收标准**：
- [ ] 模板列表展示「总量/已发/已用」和核销率
- [ ] 记录页支持按使用状态筛选
- [ ] 已使用的券可关联跳转到对应订单
- [ ] 记录支持导出

## API 接口规划

```
# 优惠券模板列表
GET /api/v1/admin/coupon-templates
  Query: { keyword?, type[], isActive? }
  Response: { items: CouponTemplate[], total: number }

# 模板详情
GET /api/v1/admin/coupon-templates/:id
  Response: CouponTemplate { ...全部字段, issuedCount, usedCount }

# 创建模板
POST /api/v1/admin/coupon-templates
  Body: CouponTemplateCreateDTO
  Response: CouponTemplate

# 编辑模板
PUT /api/v1/admin/coupon-templates/:id
  Body: CouponTemplateUpdateDTO
  Response: CouponTemplate

# 停用模板
PATCH /api/v1/admin/coupon-templates/:id/deactivate
  Response: { success: boolean }

# 发放/核销记录
GET /api/v1/admin/coupon-templates/:id/records
  Query: { keyword?, issueMethod?, status[], startDate?, endDate?, page, pageSize }
  Response: { items: UserCoupon[], total: number }

# 批量发放
POST /api/v1/admin/coupon-templates/:id/batch-issue
  Body: { target: 'all' | 'specified' | 'filtered',
          userIds?: string[],
          filter?: { storeId?, registeredAfter?, memberStatus? } }
  Response: { issuedCount: number }

# 给单个用户发放
POST /api/v1/admin/users/:userId/issue-coupon
  Body: { templateId: string, reason?: string }
  Response: { success: boolean, userCouponId: string }
```
