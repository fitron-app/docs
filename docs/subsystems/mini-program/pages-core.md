### 登录与账号初始化

> 对应主文档 §8.1、§13.1

#### 登录流程

```
用户打开小程序
    │
    ├─ 已有有效 token → 直接进入首页
    │
    └─ 无 token / token 失效
         │
         ├─ 静默调用 wx.login 获取 code
         ├─ POST /api/v1/auth/wx-login
         │
         ├─ 已注册用户 → 进入首页
         │
         └─ 新用户 → 服务端自动创建账号
              │
              ├─ 检查 faceEnrolled
              ├─ 未录脸 → 引导页（不强阻塞）
              └─ 已录脸 → 进入首页
```

#### 登录页

```
┌─────────────────────────┐
│                         │
│     🏋️ Fitron LOGO      │
│                         │
│  ╔═══════════════════╗  │
│  ║  微信一键登录      ║  │
│  ╚═══════════════════╝  │
│                         │
│  登录即表示同意           │
│  《用户协议》和《隐私政策》│
│                         │
└─────────────────────────┘
```

#### 登录异常处理

| 异常 | 处理 |
|---|---|
| `wx.login` 失败 | 展示「登录失败，请重试」+ 重试按钮 |
| token 失效（401） | 清空本地 token，重新走登录流程 |
| 拒绝授权 | 说明影响范围，保留浏览能力 |

#### 接口

```
POST /api/v1/auth/wx-login
  Body: { code: string }
  Response: {
    token: string,
    isNewUser: boolean,
    user: { id, nickname, avatarUrl, faceEnrolled: boolean }
  }
```

#### 前端质量检测

| 检测项 | 条件 | 不通过提示 |
|---|---|---|
| 正脸 | 面部角度偏转 < 15° | 「请正对摄像头」 |
| 光线 | 画面亮度在合理范围 | 「光线太暗/太亮，请调整」 |
| 清晰度 | 图像模糊度 < 阈值 | 「请保持不动，画面模糊」 |
| 遮挡 | 眼/鼻/嘴无遮挡 | 「请摘下口罩/墨镜/帽子」 |
| 多人 | 画面仅 1 张人脸 | 「请确保只有您一人在画面中」 |

> **[建议值]** 前端仅做基础质量检测，最终判断由云端 API 完成。前端通过后自动拍照上传。

#### 重录人脸（个人中心 → 人脸管理）

1. 点击「重新录入」
2. 弹出确认：「重新录入将覆盖您当前的人脸数据，确认？」
3. 确认后进入采集页
4. 上传成功后旧特征标记失效，新特征同步到工控机

#### 录脸触发场景

| 场景 | 行为 |
|---|---|
| 新用户首次登录 | 引导页（可跳过） |
| 支付成功且未录脸 | 支付结果页强引导 |
| 会员页查看且未录脸 | 黄色提醒条「未录入人脸无法入场」 |
| 个人中心 | 人脸管理入口显示状态 |

#### 接口

```
POST /api/v1/user/face/enroll
  Auth: JWT
  Body: { image: string }   // Base64
  Response: { success: boolean, message: string }

POST /api/v1/user/face/re-enroll
  Auth: JWT
  Body: { image: string }
  Response: 同上
```

### 套餐浏览与购买

> 对应主文档 §8.3、§13.2、§13.3

#### 购买流程

```
浏览套餐列表
    │
    ├─ 选择套餐 → 套餐详情（有效期、规则、退款说明）
    │
    ├─ 点击购买 → 订单确认页
    │                │
    │                ├─ 选择优惠券（可选）
    │                ├─ 金额实时刷新
    │                ├─ 点击「立即支付」
    │                │
    │                ├─ POST /api/v1/orders
    │                │    ├─ 失败 → 提示错误原因
    │                │    └─ 成功 → wx.requestPayment
    │                │
    │                ├─ 支付成功 → 轮询订单状态
    │                │    ├─ ACTIVE → 支付结果页（成功）
    │                │    └─ PAID   → 等 2s 重新轮询
    │                │
    │                └─ 取消/失败 → 订单列表（待支付订单）
    │
    └─ 待支付订单 → 继续支付 / 超时自动取消 [建议值 30min]
```

#### 套餐卡片字段

| 字段 | 说明 |
|---|---|
| 产品名称 | `name`（多语言） |
| 售价 | `price`，格式 `¥XX.XX` |
| 划线价 | `originalPrice`，删除线样式 |
| 有效期 | `durationDays` + 天 |
| 次数 | `timesTotal`（仅次卡/体验卡） |
| 标签 | `限时优惠` / `热门` / `推荐` |

#### 套餐展示规则

1. 优先通用产品（`storeId = null`），再展示门店专属
2. 按 `sortOrder` 升序
3. 体验卡仅对从未购买过任何产品的用户展示（云端判断）
4. 已下架不展示

#### 淋浴权益展示 [建议值]

> **[建议值]** 第一期：每次刷脸进入健身房自动赠送 1 次淋浴机会，每次 5 分钟。不在健身房内时不显示启动入口。

| 展示文案 | 条件 |
|---|---|
| 「启动淋浴」+ 按钮可用 | 有可用淋浴次数 |
| 「今日已用完」+ 按钮置灰 | 今日淋浴次数已用完 |
| 「请先进入健身房」+ 引导 | 未进入过健身房 |

#### 淋浴异常处理

| 异常 | 处理 |
|---|---|
| 设备离线 | 「淋浴设备暂时不可用，请联系工作人员」 |
| 无权益 | 「暂无可用淋浴次数」+ 引导进入健身房 |
| 指令成功但设备未响应 | 「淋浴启动异常，请联系客服」 |
| 网络中断 | 重新拉取状态，服务端倒计时继续执行 |

#### 交互规则

1. 输入框支持粘贴
2. 扫码调用 `wx.scanCode`，二维码内容为券码字符串
3. 兑换成功展示获得的产品和有效期
4. 失败展示具体原因：券码无效 / 已使用 / 已过期 / 活动已结束
5. 同一批次同一用户限兑换 1 张

#### 接口

```
POST /api/v1/vouchers/redeem
  Auth: JWT
  Body: { code: string }
  Response: {
    success: boolean,
    message: string,
    order?: Order       // 成功时返回生成的订单
  }
```
