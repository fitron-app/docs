# 微信小程序 — 登录与账号初始化

上级文档：[微信小程序](../mini-program)

---

### 概述

> 对应主文档 §8.1、§13.1

登录是用户使用小程序的前提。采用微信静默登录 + 服务端自动建号的方案，用户无感知即可完成身份建立。

### 登录流程

```
用户打开小程序
    │
    ├─ 已有有效 token → 直接进入首页
    │
    └─ 无 token / token 失效
         │
         ├─ 静默调用 wx.login 获取 code
         ├─ POST /api/v1/auth/wx-login
         │
         ├─ 已注册用户 → 进入首页
         │
         └─ 新用户 → 服务端自动创建账号
              │
              ├─ 检查 faceEnrolled
              ├─ 未录脸 → 引导页（不强阻塞）
              └─ 已录脸 → 进入首页
```

### 页面路由

| 路由 | 页面 | 说明 |
|:---:|---|---|
| `/pages/auth/login` | 登录授权 | 微信一键登录 + 协议确认 |

### 登录页要素

- **LOGO**：品牌标识
- **登录按钮**：调用 `wx.login` 获取 code 后提交至服务端
- **协议区域**：「登录即表示同意《用户协议》和《隐私政策》」，链接可点击查看全文
- 登录过程全自动，无需用户手动输入任何信息

### 登录异常处理

| 异常 | 处理 |
|---|---|
| `wx.login` 失败 | 展示「登录失败，请重试」+ 重试按钮 |
| token 失效（401） | 清空本地 token，重新走登录流程 |
| 拒绝授权 | 说明影响范围，保留浏览能力 |
| 网络异常 | 展示网络错误提示 + 重试按钮 |

### 登录后初始化

登录成功后，前端需并行拉取：

| 数据 | 接口 | 用途 |
|---|---|---|
| 用户资料 | `GET /api/v1/user/profile` | 展示昵称、头像、手机号 |
| 人脸状态 | `profile.faceEnrolled` | 决定是否引导录脸 |
| 会员权益摘要 | `GET /api/v1/user/membership` | 首页会员卡片、状态提醒 |
| 待支付订单 | `GET /api/v1/orders?status=PENDING` | 首页待支付提醒 |

### 接口

```
POST /api/v1/auth/wx-login
  Body: { code: string }
  Response: {
    token: string,
    isNewUser: boolean,
    user: { id, nickname, avatarUrl, faceEnrolled: boolean }
  }
```

### 线下扫码入口

用户通过线下二维码扫码进入小程序时，可能携带场景值（`scene`），需在 `onLaunch` 中解析：

| 场景值 | 来源 | 跳转行为 |
|---|---|---|
| 门店二维码 | 门店张贴 | 登录后跳转门店详情页 |
| 淋浴间二维码 | 淋浴间张贴 | 登录后跳转淋浴控制页 |
| 活动分享二维码 | 活动海报 | 登录后跳转活动详情页 |
| 通用二维码 | 无特殊参数 | 登录后进入首页 |
