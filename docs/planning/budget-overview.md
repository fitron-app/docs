# 综合预算分类说明

> **用途**：汇总 **云端高可用（南京）**、**研发协作（CODING）**、**门店硬件（单店 BOM）** 三类费用，便于做年度/立项预算。  
> **说明**：下列金额为 **参考口径**；云资源以腾讯云控制台/合同为准，CODING 以 [产品定价](https://coding.net/pricing) 及实际订购席位数为准，硬件以采购询价为准。

---

## 一、费用总览（怎么读这张表）

| 大类 | 典型计费方式 | 与门店数关系 |
|------|----------------|--------------|
| **A. 云端与网络** | 多为 **年付/包年** + 流量超出部分按量 | 与 **500 店业务负载** 相关，不按「一店一台云主机」线性增长 |
| **B. 研发工具链** | **按人/按团队** 订阅（年） | 与 **研发人数** 相关，与门店数基本无关 |
| **C. 门店硬件** | **一次性 CAPEX**（新店开业采购） | **按店数** 累加；另有少量 **单店年费**（如物联网卡） |

| 子项 | 口径 | 预估年付 (RMB) | 备注 |
|------|------|---------------:|------|
| CODING 高级版（示例） | **¥599/人/年**（与你口头 **¥600/人/年** 一致） | 按席位数累计 | 官方定价见 [CODING 产品定价](https://coding.net/pricing) |
| **按研发人数粗算** | 3 人 × ¥600 | **≈ ¥1,800** | 若允许按实际人数购买 |
| | 4 人 × ¥600 | **≈ ¥2,400** | |
| **订购规则注意** | 定价页存在 **「5 人起售、5 人一档叠加」** 类说明时 | **≈ 5 × ¥599 ≈ ¥2,995** | **下单前以控制台/合同为准**，避免预算按 3～4 人算却需买 5 席 |

**价值归纳（与自建 GitLab 对比时的预算逻辑）**：

## 四、C 类：门店硬件（单店一次性 + 单店年费）

数据来自仓库内 **[硬件 BOM](../hardware/bom.md)**（**单店、1200㎡ 基准方案**）。

| 子项 | 金额 (RMB) | 计费方式 | 说明 |
|------|-----------:|----------|------|
| **单店硬件综合采购预算** | **≈ ¥14,400** | **一次性** | 含约 10% 预留余量 |
| **单店 4G/5G 物联网卡** | **约 ¥360～¥720 / 店·年** | **年付** | BOM：¥30～¥60/月，热备流量 |
| **海康 SDK 商业授权** | 待询价 | 若有 | 以厂商报价为准 |

### 与「500 店」的关系（仅作规模感，非云端预算）

| 场景 | 计算方式 | 示例结果（仅数学演示） |
|------|----------|-------------------------|
| 新店硬件 CAPEX | 500 × ¥14,400 | **≈ ¥720 万**（若 500 店均需完整一套且同方案） |
| 物联网卡 OPEX | 500 × (¥360～¥720) | **约 ¥18 万～¥36 万 / 年** |

> 实际连锁扩张多为 **分批开店**，CAPEX 应按 **当年新开门店数** 滚动做预算，不必一次性按 500 店打满。

## 六、与架构文档的一致性说明

- 本仓库技术文档中云端 API 为 **Kotlin + Spring Boot**、数据库多为 **PostgreSQL**；预算表 A 类采用与报价一致的 PostgreSQL/Redis/托管 MQTT/CLS/备份口径，保持在同一高可用粒度下列账。
- **零停机升级**：依赖 **CLB + 双 CVM + 健康检查** 缩容/扩容权重；与 CODING 无冲突，属 **A 类运维流程**。
