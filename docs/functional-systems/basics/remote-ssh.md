# 远程 SSH 到工控机

**涉及子系统**：工控机、云端 API
**核心业务**：通过云端跳板机安全访问门店本地工控机，用于运维、排障、程序更新

---

## 架构方案

工控机位于门店内网（NAT 后），无固定公网 IP，通过反向隧道方案实现云端安全登录。

```
运维人员 → 云端跳板机（公网 IP）→ 反向隧道 → 工控机
```

推荐方案：工控机启动时建立到云端的 SSH 反向隧道（`autossh`），云端通过指定端口转发到工控机 22 端口。

---

## 安全要求

- 禁止密码登录，仅允许 SSH 密钥认证
- 跳板机仅开放特定端口（22 + 反向隧道端口段）
- 工控机 SSH 访问日志定期归档
- 反向隧道使用独立低权限系统账户运行

---

## 工控机配置要点

```bash
# /etc/systemd/system/autossh-tunnel.service
ExecStart=/usr/bin/autossh -M 0 -N \
  -o "ServerAliveInterval 30" \
  -o "ServerAliveCountMax 3" \
  -R 2222:localhost:22 \
  tunnel-user@<跳板机公网IP>
```

- [ ] 多门店是否使用不同端口段区分工控机
