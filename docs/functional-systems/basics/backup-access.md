# 数据备份与权限控制

**涉及子系统**：云端 API、管理后台
**核心业务**：数据库定时备份策略，以及管理员角色与权限体系设计

---

## 数据备份

| 备份类型 | 周期 | 保留时长 | 存储位置 |
|---|---|---|---|
| 全量备份 | 每日凌晨 3 点 | 30 天 | 对象存储（异地） |
| 增量备份 | 每小时 | 7 天 | 对象存储 |
| 人脸特征向量 | 每日同步 | 永久 | 对象存储（加密） |

备份文件使用 AES-256 加密，解密密钥由 KMS 管理。

---

## 管理员角色体系

| 角色 | 权限范围 | 说明 |
|---|---|---|
| 超级管理员 | 全部 | 系统配置、用户管理、所有门店数据 |
| 老板 | 所有门店数据查看 + 财务 | 只读为主，可审批退款 |
| 财务 | 所有门店财务数据 | 营收、结算、退款报表 |
| 门店管理员 | 仅本店数据 | 设备控制、用户管理、本店订单 |
| 运维人员 | 工控机管理、OTA | 无业务数据访问权限 |

---

## 权限实现

- 基于 RBAC（角色权限控制）
- 角色与门店绑定：门店管理员仅能操作绑定门店
- API 层面统一鉴权中间件，不依赖前端隐藏控件
- 操作日志：所有写操作记录操作人、时间、变更内容
