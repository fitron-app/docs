# API 定义文档系统

**涉及子系统**：云端 API
**核心业务**：基于 OpenAPI/Swagger 自动生成接口文档，统一前端、工控机、第三方对接的接口规范

---

## 方案

云端 API 基于 Kotlin + Spring Boot，使用 SpringDoc（OpenAPI 3.0）自动生成接口文档。

- **文档地址**（内网）：`https://api-internal.fitron.com/swagger-ui.html`
- **OpenAPI JSON**：`/v3/api-docs`

---

## 文档分组

| 分组 | 路径前缀 | 适用对象 |
|---|---|---|
| 用户端 | `/api/v1/` | 小程序客户端 |
| 管理端 | `/api/v1/admin/` | 管理后台 |
| 工控机端 | `/api/v1/ipc/` | 工控机 |
| 支付回调 | `/api/v1/pay/` | 微信支付服务器 |

---

## 规范要求

- 所有接口必须有中文描述
- 请求/响应字段必须有注释说明
- 错误码统一在文档中列举
- 接口变更前需更新文档并通知相关开发人员
